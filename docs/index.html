雙協定 · Dual Protocol

同一個伺服器，兩套標準。

不管你手上的客戶端走 OpenAI 還是 Anthropic 協定，指向同一個 Windsurf API 實例就能用。

S

OpenAI 相容

POST /v1/chat/completions 
$ curl http://localhost:3003/v1/chat/completions \
  -H "Authorization: Bearer sk-ws-demo" \
  -d '{
    "model": "gpt-5.2",
    "messages": [
      {"role":"user","content":"hello"}
    ],
    "stream": true
  }'
# → Server-Sent Events (OpenAI chunk 格式)
A

Anthropic 相容

POST /v1/messages 
$ curl http://localhost:3003/v1/messages \
  -H "x-api-key: sk-ws-demo" \
  -d '{
    "model": "claude-opus-4.6",
    "messages": [
      {"role":"user","content":"hello"}
    ],
    "stream": true
  }'
# → SSE (Anthropic message_delta 格式)
同一個請求體格式、同一個帳號池、同一個速率限制器——切換協定只是換路由前綴。Claude Code、Cline 吃 /v1/messages，Cursor/OpenAI SDK 吃 /v1/chat/completions，互不干擾。
模型 · Models

一個 API 背後，107 個模型。

從 Claude Opus 到 GPT-5、從 Gemini 3 到 DeepSeek R1，9 個供應商的模型統一接入。點標籤即時篩選。

免費帳號：僅支援 gpt-4o-mini 與 gemini-2.5-flash。其餘模型需 Windsurf Pro 訂閱。
看不到最新模型？你的 Language Server binary 可能落後 Windsurf 雲端，從桌面端拷貝最新版 LS 覆蓋即可——/v1/models 會自動從雲端發現新模型。
架構 · Architecture

中間多一層，全都接住。

請求進來後經過協定翻譯、帳號池選號、語言伺服器轉 gRPC，最後到 Windsurf 雲端。速率限制、故障轉移、身份剝離在中間層完成。

部署 · Deploy

從零到跑起來，五分鐘。

Linux 伺服器 · Node.js ≥ 20 · 零 npm 依賴。以下所有步驟按序執行即可。

安裝 Node.js 20+

Ubuntu / Debian 的最快方式：

bash
# NodeSource 官方源
$ curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
$ apt install -y nodejs

# 驗證
$ node -v   # v20.x+

Clone 專案

bash
$ git clone https://github.com/dwgx/WindsurfAPI.git
$ cd WindsurfAPI

安裝 Language Server

倉庫自帶 install-ls.sh：自動下載 Exafunction release、放到 /opt/windsurf/、chmod 可執行。

bash
# 一鍵安裝（自動下載最新版）
$ bash install-ls.sh

# 或指定 URL
$ bash install-ls.sh --url https://example.com/language_server_linux_x64

# 或用本地已有檔案
$ bash install-ls.sh --file ./language_server_linux_x64

設定 .env

複製範本後修改：

.env
PORT=3003
API_KEY=                   # 留空 = 不驗證
DEFAULT_MODEL=gpt-4o-mini
MAX_TOKENS=8192
LOG_LEVEL=info
LS_BINARY_PATH=/opt/windsurf/language_server_linux_x64
LS_PORT=42100
DASHBOARD_PASSWORD=         # 留空 = 後台免密碼

啟動（PM2 守護）

bash
$ npm install -g pm2
$ pm2 start src/index.js --name windsurf-api
$ pm2 save && pm2 startup   # 開機自啟
重啟請勿用 pm2 restart。會留殭屍進程佔住端口。正確做法： pm2 stop && pm2 delete && fuser -k 3003/tcp && pm2 start——或直接跑 bash update.sh。

新增帳號

從 Windsurf 編輯器取得 token 後：

bash
# 單個
$ curl -X POST http://localhost:3003/auth/login \
    -H "Content-Type: application/json" \
    -d '{"token":"your-windsurf-token"}'

# 批次
$ curl -X POST http://localhost:3003/auth/login \
    -H "Content-Type: application/json" \
    -d '{"accounts":[{"token":"t1"},{"token":"t2"}]}'

驗證

bash
$ curl http://localhost:3003/health

$ curl http://localhost:3003/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hi"}]}'

# 瀏覽器開啟管理後台
# http://YOUR_IP:3003/dashboard

常見問題 · FAQ

你可能想問的。

需要 Windsurf 付費帳號嗎？
免費帳號可以跑，但僅限 gpt-4o-mini 和 gemini-2.5-flash。Claude、GPT-5 全系、Gemini 3 等都需要 Windsurf Pro 訂閱。後台會自動偵測並標記每個帳號的 tier。
可以 Windows 上跑嗎？
HTTP 伺服器和管理後台能跑，但 Language Server binary 只有 Linux 版本（language_server_linux_x64），所以聊天功能僅限 Linux。Windows 建議放 WSL2 或純 Linux VM。
看不到最新模型（opus-4.7、gpt-5.3 之類）怎麼辦？
Exafunction 公開 release 停在 v2.12.5（2026-01），不含 4.7。從 Windsurf 桌面端 App 裡把 LS binary 拷出來即可：
• macOS：~/Library/Application Support/Windsurf/resources/app/extensions/windsurf/bin/language_server_macos_arm
• Linux：~/.windsurf/bin/language_server_linux_x64
拷過去後 /v1/models 會自動從雲端 discover 最新目錄。
帳號會被封嗎？
Cascade 流量型態與桌面端不一樣，高頻 burst 確實容易被識別。後台的封禁偵測面板會監控錯誤率，並按 5 分鐘窗口做速率冷卻。推薦：每帳號 RPM 控制在 10 以下、配不同出口代理、混用思考型和普通模型。
和 pqhaz3925/AlexStrNik 那些專案的區別？
三點核心差異：
(1) 雙協定——那些是 OpenAI only，本專案同時有 /v1/messages。
(2) planner_mode=NO_TOOL——關掉 Cascade 內建工具循環，消除 20% stall warm 假陽性和路徑洩露。
(3) 完整管理後台 + 帳號池——不是單號代理，是多號輪詢 + 故障轉移。
是 MIT License 嗎可以商用嗎？
代碼本體 MIT License，法律上允許商用。README 顶上有一段作者態度：沒給 Star 和 Follow 的請別商業轉售，點了的隨便用。請自行判斷要不要尊重這個態度。
